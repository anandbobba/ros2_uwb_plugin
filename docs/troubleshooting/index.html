Troubleshooting

Build Issues​

colcon build fails with missing packages​

ERROR: Package 'ros2_uwb_msgs' not found

Fix: Run rosdep install before building:

cd ~/ros2_uwb_ws
rosdep install --from-paths src --ignore-src -r -y
colcon build --symlink-install

CMake Error: Could not find a package configuration file for "ignition-gazebo6"​

Fix: Install Ignition Fortress:

sudo apt install ros-humble-ros-gz

Python import errors in analysis scripts​

ModuleNotFoundError: No module named 'pandas'

Fix:

pip3 install pandas matplotlib scipy

Simulation Issues​

[WARNING] Jump in time detected​

Gazebo is running slower than real-time. The simulation clock is jumping.

Fix: Run in headless mode:

ros2 launch ros2_uwb_research_sim demo.launch.py gazebo_gui:=false

Or reduce physics update rate in world.sdf:

<physics>
  <max_step_size>0.004</max_step_size>
  <real_time_factor>1.0</real_time_factor>
</physics>

Robot does not move after clicking Play​

Fix: Make sure you clicked Play (▶) in Gazebo, not just launched the simulation. The autopilot waits for the simulation clock to start.

/uwb/pose not publishing​

Check the preprocessor has enough valid ranges:

ros2 topic hz /uwb/range
ros2 topic hz /uwb/ranges_filtered

If /uwb/ranges_filtered is silent, the preprocessor is dropping all measurements. Common causes:

  • Fewer than 3 anchors publishing
  • Outlier threshold too strict (check preprocessor.yaml)

Communication Issues​

FastDDS discovery failures (topics not visible across machines)​

Fix: Set a shared discovery server or use a simple peer config:

export RMW_IMPLEMENTATION=rmw_fastrtps_cpp
export FASTRTPS_DEFAULT_PROFILES_FILE=~/fastdds_simple.xml

Or switch to CycloneDDS:

sudo apt install ros-humble-rmw-cyclonedds-cpp
export RMW_IMPLEMENTATION=rmw_cyclonedds_cpp

Hardware Issues​

No data from serial port​

# Check device is visible
ls /dev/ttyUSB*

# Check permissions
stat /dev/ttyUSB0

# Grant access
sudo chmod 666 /dev/ttyUSB0

If the device disappears after reconnect, add a udev rule:

sudo nano /etc/udev/rules.d/99-uwb.rules
# Add: SUBSYSTEM=="tty", ATTRS{idVendor}=="0483", MODE="0666"
sudo udevadm control --reload-rules

Ranges look correct but pose is noisy / diverging​

  • Check that anchor positions in uwb_driver.yaml match your physical anchor layout exactly
  • Ensure all anchors are publishing at similar rates: ros2 topic hz /uwb/range
  • Try reducing the EKF process_noise_covariance in ekf.yaml for smoother output
